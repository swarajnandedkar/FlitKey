Operating System Compatibility

TypeFlux is designed for desktop systems. We believe in platform transparency and clear support capabilities.

OS / Platform Expansion Style System Tray & UI Setup Requirements Status
Windows 10 / 11 Full Typed Expansion Supported (System Tray) Executable installer, no Python dependency required Gated Beta
Linux (X11 Desktop) Full Typed Expansion Supported (System Tray) Requires xdotool, xclip installed Production Ready
Linux (Wayland Desktop) Quick Insert & Clipboard Desktop specific / App Window Requires wl-clipboard for expansion Partial Support

💡 Wayland Note: Due to security designs in Wayland preventing global key-listening, TypeFlux defaults to a Quick Insert bar (Raycast prompt style) which copies your selected snippet to the clipboard and pastes it automatically.

Frequently Asked Questions

Everything you need to know about security, licenses, and compatibility.

To replace a shortcut (e.g. :date) system-wide, TypeFlux must listen for typed characters. When a registered trigger is matched, TypeFlux deletes the trigger inputs using keyboard event simulation and pastes or types the expanded content. It operates completely offline, and your keystrokes never leave your machine.

No. TypeFlux stores snippets, history, and preferences locally in plain-text JSON within your local system directory (e.g., ~/.config/typeflux/ on Linux or AppData/Roaming/TypeFlux on Windows). We do not run synchronization servers, nor do we require an account.

Wayland restricts direct system-wide key monitoring for security reasons. Therefore, on Wayland sessions, TypeFlux uses a dedicated "Quick Insert" command bar (Raycast style). Pressing the hotkey opens a quick selection bar where you search, press Enter, and the snippet is immediately copied and inserted using the clipboard.
